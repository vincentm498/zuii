# Internationalisation (i18n) dans Zuii

Cette documentation explique comment fonctionne le système multilingue dans les composants Zuii et comment ajouter ou modifier des langues.

## Architecture

Le système i18n repose sur trois piliers :
1.  **Le type global** : `ZuiiLang` définit les codes de langues supportés à l'échelle du projet.
2.  **Les dictionnaires par module** : Chaque package (Booking, Calendar, etc.) possède son propre fichier `src/trads/i18n.ts`.
3.  **Le dictionnaire centralisé** : Un objet `trads` regroupe toutes les langues pour un accès dynamique simple.

---

## Guide d'utilisation

### 1. Structure du fichier `i18n.ts`

Chaque module doit exporter un objet `trads`. Voici la structure recommandée :

```typescript
// packages/mon-module/src/trads/i18n.ts
export const fr = {
    title: 'Bonjour',
    // ... autres clés
};

export const en = {
    title: 'Hello',
};

// L'export principal utilisé par le composant
export const trads = { fr, en };
```

### 2. Intégration dans un composant JS/TS

Dans votre classe de composant, vous devez charger les traductions en fonction de l'option `lang` :

```typescript
import { trads } from '../trads/i18n';
import { type ZuiiLang } from '@zuii/core';

class MonComposant {
    private trads: any;

    constructor(options: { lang: ZuiiLang }) {
        // Sélection dynamique sans if/else
        this.trads = trads[options.lang] || trads.fr;
    }

    public setLanguage(lang: ZuiiLang) {
        this.trads = trads[lang] || trads.fr;
        this.render(); // Redessiner le composant
    }
}
```

## Ajouter une nouvelle langue

Pour ajouter une nouvelle langue (ex: l'italien `it`) :

1.  **Mettre à jour le type global** :
    Dans `packages/core/src/utils/lang_types.ts`, ajoutez le code de la langue :
    ```typescript
    export type ZuiiLang = 'fr' | 'en' | 'es' | 'it';
    ```

2.  **Traduire les modules** :
    Dans chaque module (`booking`, `calendar`, etc.), ouvrez `src/trads/i18n.ts` :
    -   Créez une constante `it = { ... }` avec toutes les clés requises.
    -   Ajoutez `it` dans l'objet `trads` final : `export const trads = { fr, en, es, it };`.

3.  **Locales externes (ex: date-fns)** :
    Si le module utilise une bibliothèque tierce comme `date-fns`, n'oubliez pas d'importer la locale correspondante dans le composant principal (ex: `import { it as localeIt } from 'date-fns/locale'`).

---

## Bonnes Pratiques

-   **Évitez les textes en dur** : Aucun texte affiché à l'utilisateur ne doit être écrit directement dans le fichier `.ts` du composant. Tout doit passer par `this.trads`.
-   **Centralisation des labels de champs** : Pour les formulaires (ex: Booking), les labels par défaut (Nom, Prénom) doivent être définis dans le fichier `i18n.ts` sous une clé `fields`.
-   **Repli (Fallback)** : Utilisez toujours une valeur par défaut (généralement `trads.fr`) si une langue n'est pas trouvée pour éviter les erreurs de rendu.

---

## Désactiver une langue

Si vous souhaitez désactiver temporairement ou définitivement une langue du projet (ex: désactiver l'espagnol `es`) :

1.  **Désactivation globale** :
    Rendez-vous dans `packages/core/src/utils/lang_types.ts` et retirez la langue du type **ET** de la constante :
    ```typescript
    export type ZuiiLang = 'fr' | 'en'; // 'es' retiré
    export const SUPPORTED_LANGS: ZuiiLang[] = ['fr', 'en']; // 'es' retiré
    ```
    *Cela garantira que les futurs composants ne proposeront pas cette langue et que TypeScript signalera toute erreur si elle est encore utilisée.*

2.  **Nettoyage des modules** (Optionnel) :
    Vous pouvez retirer la constante `es` de chaque fichier `src/trads/i18n.ts` pour alléger le bundle final, mais ce n'est pas strictement obligatoire pour la désactivation fonctionnelle.
  
3.  **Filtrage UI** :
    Les sélecteurs de langue ou composants dynamiques doivent utiliser `SUPPORTED_LANGS` pour générer leurs options, ce qui leur permet de s'adapter automatiquement aux modifications du point 1.

## Surcharger les traductions (Projet spécifique)

Il est possible de modifier les textes ou d'ajouter une langue "à la volée" sans modifier la bibliothèque Zuii en utilisant l'option `labels`.

Cette option fusionne les textes fournis avec ceux de la langue actuelle. Si une clé est présente dans `labels`, elle gagne la priorité.

### Exemple : Ajouter l'Allemand sans modifier le cœur

Si votre projet a besoin de l'allemand mais que Zuii ne le supporte pas nativement :

1.  **Dans votre template (ex: Twig)** :
    ```twig
    {% set german_labels = {
        'selectDay': 'Bitte wählen Sie einen Tag',
        'confirmTitle': 'Buchung bestätigen',
        'fields': {
            'firstname': 'Vorname',
            'lastname': 'Nachname'
        }
    } %}

    <div class="booking" data-labels="{{ german_labels|json_encode }}"></div>
    ```

2.  **Dans votre fichier JavaScript** :
    ```javascript
    const labels = JSON.parse(el.dataset.labels || '{}');

    new Booking(el, {
        labels: labels, // Les labels allemands écraseront le français par défaut
        // ...
    });
    ```

### Pourquoi utiliser cette méthode ?

-   **Zéro modification du cœur** : Gardez votre bibliothèque Zuii propre et légère.
*   **Rapidité** : Idéal pour des ajustements ponctuels de libellés ou pour supporter une langue rare sur un projet spécifique.
*   **Puissance** : Vous pouvez utiliser le système de traduction de votre framework backend (Symfony, Laravel, etc.) pour remplir l'objet `labels`.
